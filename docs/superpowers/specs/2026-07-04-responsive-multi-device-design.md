# 多端响应式适配设计（Phase 1：全局断点与导航）

## 背景

Class-Lab（品牌名「士多啤梨」）目前是固定 `max-w-4xl` 定宽布局的 React 单页应用，未做响应式适配。项目现有 4 大模块：

- **DTT 教案** / **视频分析**：主要由老师在 PC 上使用（备课、写教案、看分析报告）。
- **绘本打卡** / **游戏乐园**：主要由孩子在平板/手机上使用（现场互动、玩游戏）。

目标：把项目改造为响应式网页，同时兼顾 PC、平板、手机三种设备，且不同模块可以按其主要使用场景优化布局，而不是所有设备用完全一样的布局。

## 范围

本设计文档只覆盖 **Phase 1：全局断点规则与顶层导航（AppShell）+ 容器宽度策略**。后续按模块分阶段推进（不在本文档范围内）：

- Phase 2：绘本打卡 + 游戏乐园（孩子端优先，平板/手机交互细节，如翻页书、拖拽类游戏的触屏适配）。
- Phase 3：DTT 教案 + 视频分析（老师端优先，PC 宽屏利用、表单/详情页排版）。

## 技术方向

响应式网页：沿用现有 React + Vite + Tailwind 单应用架构，通过 Tailwind 断点类适配不同设备，不引入设备检测、不拆分独立 App/PWA。

## 断点规则

沿用 Tailwind 默认断点，不新增自定义断点：

| 断点 | 宽度 | 对应设备 |
| --- | --- | --- |
| （默认） | < 640px | 手机 |
| `sm` | ≥ 640px | 大屏手机/竖屏小平板 |
| `md` | ≥ 768px | 平板竖屏 |
| `lg` | ≥ 1024px | 平板横屏/小尺寸 PC |
| `xl` | ≥ 1280px | PC 宽屏 |

导航切换以 `md`（768px）为界：`md` 以下视为「移动端」（手机为主），`md` 及以上视为「桌面端」（平板横屏/PC 为主）。

## 顶层导航（AppShell.tsx）

- **`md` 及以上**：保持现状——顶部 sticky 栏内展示 logo + 横向 Tab（DTT 教案/绘本打卡/游戏乐园/视频分析）+ 用户信息。
- **`md` 以下**：
  - 顶部栏简化为 logo + 用户信息，去掉横向 Tab（宽度不够放 4 个 Tab + 文字）。
  - 新增底部固定 Tab 栏（`fixed bottom-0`），4 个 Tab 各配一个 emoji 图标 + 文字：
    - 📋 DTT 教案
    - 📖 绘本打卡
    - 🎮 游戏乐园
    - 🎥 视频分析
  - 每个 Tab 点击热区高度 ≥ 44px（移动端触摸可用性基本要求）。
  - 底部栏加 `padding-bottom: env(safe-area-inset-bottom)` 适配 iOS 带 Home Indicator 的机型。
  - `<main>` 在 `md` 以下增加底部 padding（约等于底部栏高度 + 安全区），避免页面内容被固定栏遮住。

当前的 Tab 匹配逻辑（`tabs` 数组的 `match` 函数）复用，不改变路由结构。

## 容器宽度策略

不引入统一的布局组件封装，按页面类型直接调整 Tailwind 类（保持现有 `max-w-4xl mx-auto` 写法的页面级别修改）：

- **表单/详情类页面**（`NewLesson`、`LessonDetail`、`VideoAnalysisDetail`）：保持 `max-w-4xl` 不变——这类内容以阅读/填表为主，过宽会降低可读性。
- **网格/列表类页面**（`Home` 教案列表、`PictureBook` 打卡列表、`Games` 游戏入口）：容器宽度改为 `max-w-4xl lg:max-w-6xl xl:max-w-7xl`，让 PC 宽屏下网格能展示更多列，避免内容局促在页面中间一小块。

## 不在本阶段做的事

- 不处理各模块内部的具体交互适配（游戏画布、翻页书触屏手势、视频上传控件等），留给 Phase 2/3。
- 不做设备类型检测或分别渲染完全不同的组件树。
- 不引入 CSS-in-JS 或额外的响应式工具库。

## 验证方式

无自动化测试（前端后续切片不写测试，见项目约定）。改完后用浏览器在以下宽度截图核实：

- 375px（手机竖屏）
- 768px（平板竖屏）
- 1024px（平板横屏）
- 1440px（PC 宽屏）

检查点：导航栏在断点处正确切换、底部 Tab 不遮挡内容、触摸热区足够大、网格列数随宽度变化、无横向滚动条溢出。
