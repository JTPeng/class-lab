# 个案闭环验证报告

个案建档 → 教案生成 → 执行记录 → 家长反馈 → 调整建议 闭环功能验证。

## 一、验证结论

| 环节 | 结果 |
|---|---|
| 教师建档、生成教案 | 通过 |
| RoundFlow 执行记录+打分 | 通过 |
| 个案详情正确率趋势 | 通过 |
| 家长免登录分享页查看+反馈 | 通过 |

## 二、教师端验证步骤

1. 登录后默认进入「DTT 个案」页（首页），点击「＋ 新建个案」，填写姓名/别称与能力评估基线，创建后应出现在个案列表中。
2. 点击进入该个案，点击「＋ 新建教案」，填写目标技能（如"认识动物"）等信息生成教案，生成完成后自动跳转到教案详情页。
3. 在教案详情页的「教学流程（回合演示）」区域，点击「孩子正确」/「孩子错误」若干次，页面下方"本次已记录 X 次正确 / Y 次错误"应实时更新。
4. 点击「结束训练」，填写配合度、阶段性进步印象后点击「提交打分」，应看到「本次训练记录已提交」提示。
5. 返回个案详情页，应看到：
   - 「正确率趋势」图表新增一根柱子，对应本次正确率。
   - 累计满 3 次执行记录后，若最近 3 次正确率均 ≥90%（且家长反馈均为"偏易"）或均 ≤50%（或家长反馈"偏难"），页面顶部会出现橙色调整建议文案。
6. 点击「家长分享」，弹出二维码与链接（形如 `http://<局域网IP>:<端口>/share/<token>`），可用手机扫码或复制链接发给家长。

## 三、家长端验证步骤（无需登录）

1. 打开教师提供的分享链接（或扫码），应直接看到个案姓名、基线信息，无需任何登录操作。
2. 「训练记录」列表按时间倒序展示每次执行记录的正确率（如"1/2 正确"）。
3. 对于还没有反馈的记录，会显示反馈表单：选择难度（偏易/刚好/偏难）、拖动兴趣投入度滑杆（1-5 分）、可选填写留言，点击「提交反馈」。
4. 提交后该条记录变为只读展示："难度反馈：XX · 兴趣投入度：X 分"，不可重复提交。
5. 回到教师端个案详情页刷新，该条记录的反馈应已同步；累计反馈满足规则条件时，调整建议文案会相应更新。

## 四、回归检查

确认「绘本打卡」「游戏乐园」两个模块页面正常打开、功能未受本次改动影响。

## 五、实测数据（本次验证用例）

- 个案：小明（能理解简单指令，配对能力弱）
- 教案：认识动物 DTT 训练方案
- 执行记录：4 次，正确率分别为 50% / 100% / 90% / 100%
- 家长反馈：后 3 次均标记"偏易"，兴趣投入度 3-4 分
- 触发文案：*"最近 3 次正确率均在 90% 以上，家长反馈偏易，建议提高目标难度或进入下一阶段。"*
